# Рецепт: Git-коммиты в Brain

## Назначение

Анализ истории git-репозитория и построение графа знаний из коммитов, авторов, файлов и изменений. Позволяет искать по истории коммитов на естественном языке (например, "когда мы меняли auth middleware?").

## Предварительные требования

- [ ] Levara-сервер запущен (`curl http://localhost:8080/health`)
- [ ] LLM настроен (`LLM_ENDPOINT` + `LLM_MODEL` установлены)
- [ ] Git-репозиторий доступен на локальной файловой системе
- [ ] `doctor()` показывает LLM status = "ok"

## Шаги

1. **Установить контекст** для целевого проекта:
   ```
   set_context(collection="myproject")
   ```

2. **Анализ коммитов** — извлекает сущности из сообщений коммитов и диффов:
   ```
   analyze_commits(
     repo_path="/path/to/repo",
     since="2026-01-01",
     limit=100
   )
   ```

   Запускает cognify-пайплайн на каждом коммите, извлекая:
   - Сущности авторов (person nodes)
   - Сущности файлов/модулей
   - Описания изменений как chunks
   - Связи (AUTHORED, MODIFIED, DEPENDS_ON)

3. **Дождаться обработки** — проверить статус:
   ```
   cognify_status(run_id="<returned_id>")
   ```

4. **Проверить загрузку**:
   ```
   search(search_query="последние изменения в аутентификации")
   ```

## Smoke-тест

```
git_search(query="кто работал над поиском?")
```

Ожидание: результаты с упоминанием авторов коммитов и релевантных изменений файлов.

```
query_entity(name="<имя_автора>")
```

Ожидание: сущность с AUTHORED рёбрами к коммитам.

## Откат

Удалить dataset, созданный analyze_commits:
```
delete(dataset_id="<dataset_id>")
```

## Заметки

- `analyze_commits` использует LLM для извлечения сущностей — если LLM через API, есть затраты
- Большие репозитории: используй `since` и `limit` для контроля объёма. Начни с последних 50-100 коммитов.
- Повторный запуск на том же репозитории создаст дублирующие сущности если не удалить предыдущие
- Для инкрементальных обновлений используй `since` с датой последнего анализа
