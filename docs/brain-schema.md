# Brain Schema

Рекомендуемая структура знаний для организации memories и collections в Levara.

## Коллекции

Одна коллекция на проект или домен:

| Коллекция | Назначение |
|-----------|-----------|
| `levara` | Знания проекта Levara |
| `work` | Рабочие знания (кросс-проектные) |
| `personal` | Личные заметки, контакты, предпочтения |
| `<имя-проекта>` | Любой другой проект |

## Rooms (подтемы внутри коллекции)

Стандартные rooms — используй единообразно во всех коллекциях:

| Room | Что сюда |
|------|---------|
| `people` | Информация о конкретных людях |
| `companies` | Факты о компаниях/организациях, культура, продукты |
| `projects` | Статус проекта, scope, deliverables |
| `meetings` | Заметки с встреч, action items, решения |
| `decisions` | Архитектурные и проектные решения с обоснованием |
| `infrastructure` | Серверы, сервисы, порты, деплой |
| `codebase` | Неочевидные паттерны кода, gotchas, соглашения |
| `bugs` | Расследования багов, root causes, фиксы |
| `ideas` | Планы на будущее, брейнштормы, исследования |

Кастомные rooms приветствуются — список выше это отправная точка, не ограничение.

## Halls (жанр памяти)

Контролируемый словарь — эти значения обязательны:

### `fact`
Объективная характеристика. Версионируемая, с таймстампом.

```
key: "levara-embed-dim"
value: "Levara использует nomic-embed-text-v2-moe с 768 измерениями"
room: "infrastructure"
hall: "fact"
```

### `event`
Что-то произошло. Всегда с абсолютной датой.

```
key: "merge-freeze-2026-03-05"
value: "Мерж-фриз начался 2026-03-05 для релиза мобильной версии"
room: "projects"
hall: "event"
```

### `decision`
Архитектурный или проектный выбор. Всегда с **почему**.

```
key: "auth-jwt-not-session"
value: "Используем JWT (не session cookies) для API auth потому что: stateless, на Pi нет Redis, проще горизонтальное масштабирование"
room: "decisions"
hall: "decision"
```

### `preference`
Предпочтение пользователя по стилю/инструменту/workflow. Пинить если глобальное.

```
key: "user-prefers-terse"
value: "Пользователь предпочитает краткие ответы без итоговых саммари"
room: "people"
hall: "preference"
pin: true, priority: 10
```

### `advice`
Переиспользуемая рекомендация или best practice.

```
key: "qwen-no-think"
value: "Qwen 3.5 требует /no_think префикс + num_predict 3000-5000 чтобы не уходить в thinking-mode"
room: "infrastructure"
hall: "advice"
```

### `discovery`
Неочевидный инсайт, баг или подводный камень.

```
key: "cognify-variable-shadowing"
value: "dedupResult := внутри if блока создавал scoped переменную — внешняя оставалась пустой, Neo4j записал 0 нод. Фикс: := на ="
room: "bugs"
hall: "discovery"
```

## Теги

Опциональные метаданные для сквозных тем:

| Тег | Для чего |
|-----|---------|
| `#critical` | Must-know факты, высокий приоритет |
| `#wip` | В процессе, может измениться |
| `#archived` | Больше не активно, сохранено для истории |
| `#reference` | Указатель на внешний ресурс |
| `#security` | Информация связанная с безопасностью |
| `#performance` | Находки по производительности |

Теги позволяют фильтрованный поиск: `search(search_query="auth", tags=["security"])`

## Гайд по приоритету пинов

| Приоритет | Для чего |
|-----------|---------|
| **10** | Глобальные предпочтения пользователя (стиль, язык, правила) |
| **8** | Критическая инфраструктура (endpoints, IPs, порты, версии) |
| **5** | Активные решения по проекту |
| **3** | Полезный, но некритичный контекст |
| **1** | Низкий приоритет, nice-to-have |

Держи запиненную память компактной — `wake_up` имеет бюджет по токенам. Открепляй устаревшие записи.
