# Levara Solo Pro — одна память на все ваши устройства

**Mac, домашний сервер, Raspberry Pi — память одна, и она всегда с вами.**

## Для кого

Опытный пользователь, у которого **несколько машин**: рабочий ноутбук, домашний
компьютер, мини-сервер (например, Raspberry Pi). Хочется, чтобы ИИ-память была
единой везде.

## Боль

- Память «привязана» к одной машине — на другой её нет.
- Нет резервных копий: сбой диска — и наработки потеряны.
- Хочется лёгкого контроля: что происходит, сколько данных, всё ли живо.

## Что даёт Solo Pro

Всё из уровня Personal, плюс **синхронизация между устройствами**, резервные
копии и базовые метрики.

| Возможность | Что это даёт вам |
|---|---|
| Синхронизация Mac ↔ сервер | Память следует за вами между устройствами |
| Защищённый токен синхронизации | Обмен данными только между вашими доверенными узлами |
| SQLite или PostgreSQL, локально или S3 | Гибкий выбор хранилища под ваш сетап |
| Резервные копии и восстановление | Сбой оборудования больше не означает потерю памяти |
| Метрики (Prometheus) | Видно состояние системы — без сложного администрирования |

## Когда переходить с Personal

Переходите на Solo Pro, когда память уже стала рабочим инструментом и её нельзя
держать только на одном ноутбуке. Типичные сигналы: появился домашний сервер,
нужны ночные бэкапы, хочется использовать тот же контекст на Mac и Pi.

## Сценарий

Днём вы работаете на Mac, вечером — на домашнем мини-сервере. Решение, записанное
за рабочим столом, уже доступно дома: синхронизация подхватила его автоматически.
А ночной бэкап гарантирует, что даже при сбое диска ничего не пропадёт.

## Доказательства в продукте

- `LEVARA_PROFILE=solo_pro` допускает SQLite или Postgres.
- Sync использует bearer token (`LEVARA_TOKEN`) и предупреждает о version skew.
- `make test-release-candidate` включает sync и backup/restore gates.
- S3-compatible raw storage доступен как optional adapter, без требования к
  Personal/local setup.

## Почему Levara

- **Единый контекст везде** — без ручного переноса файлов.
- **Спокойствие** — резервные копии и восстановление по понятным сценариям.
- **Прозрачность** — лёгкие метрики вместо тяжёлого мониторинга.
- **Тот же продукт** — апгрейд с Personal не требует миграции на что-то другое.

## Как начать

```bash
cp deploy/profiles/solo_pro.sync.env.example .env
# указать адрес второго узла и стабильный токен синхронизации
./levara-server -config-check
```

В строгом режиме (`LEVARA_PROFILE_STRICT=1`) система сама напомнит, если
синхронизация настроена без токена.

## CTA для лендинга

**Соберите личную memory-сеть.** Запустите первый узел на Mac, второй на Pi или
домашнем сервере, включите sync token и проверьте восстановление из бэкапа.
