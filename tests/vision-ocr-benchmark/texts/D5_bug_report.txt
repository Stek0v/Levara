Bug: SQLite "database is closed" after first write. Root cause: fasthttp calls io.Closer.Close() on all values stored via c.Locals() when the request context is recycled (valyala/fasthttp/userdata.go:77-78). The middleware c.Locals("auth_db", pgDB) stored *sql.DB directly — fasthttp called pgDB.Close() after the first request, killing the entire connection pool permanently. Fix: wrap *sql.DB in DBRef struct that does not implement io.Closer.
